## 🧠 장기기억 시스템

### 설계 방식

| 타입 | 역할 | 최대 개수 | 프롬프트 주입 방식 |
|------|------|-----------|------------------|
| core_concept | 절대 기억 (연애시작/결혼/죽음 등 서사 전환점) | 5개 | 매 턴 무조건 전부 포함 |
| timeline | 사건 타임라인 | 80개 | RAG 방식 - 키워드 매칭 시만 최대 5개 |
| relationship | 캐릭터 관계도 (유저 직접 입력) | 캐릭터별 JSON 배열 | 매 턴 무조건 포함 |

### 관계도 저장 형식 (JSON 배열)
```json
[
  {
    "character_name": "철수",
    "emotion": "집착하는 사랑",
    "relationship": "연인"
  }
]
```

### buildMemoryPrompt 출력 형식
```
[Memory]
Core: 내용1 / 내용2
Timeline: [턴범위] 내용 | [턴범위] 내용
Relationship:
- 철수 | 유저를 향한 감정: 집착하는 사랑 | 사이: 연인
```

### 기억 추출
- **추출 모델**: `gemini-2.0-flash-lite` (또는 flash-lite-preview)
- **추출 주기**: 5턴마다
- **relationship 타입**: AI 자동 추출 제거 → 유저 직접 입력 방식
- **DB 조회 limit**: 100개
- **타임라인 추출**: lastRecordedTurn 이후 새 사건만 추출
